## Build it yourself
```bash
docker build -t my-comfy-workflow .
docker run --rm --gpus all -p 8188:8188 my-comfy-workflow
```
If the workflow uses gated HuggingFace or CivitAI models, pass tokens at build time:
```bash
docker build \
  --build-arg HF_TOKEN=$HF_TOKEN \
  --build-arg CIVITAI_API_TOKEN=$CIVITAI_API_TOKEN \
  --build-arg CIVITAI_API_KEY=$CIVITAI_API_KEY \
  -t my-comfy-workflow .
```
`CIVITAI_API_TOKEN` is the preferred variable for `comfy model download`. `CIVITAI_API_KEY` is still accepted for compatibility.

## Deploy on Runpod
1. Connect this repository at https://runpod.io/console/serverless
2. Create a new endpoint, select **Deploy from GitHub**
3. Pick this repo, branch `main`
4. Runpod's builder will build the Dockerfile and host the resulting image
5. Hit the endpoint with the API workflow JSON (`api-workflow.json` in this repo)

## Runtime model source (recommended)
This repo now downloads the checkpoint at container startup (not at docker build time).

It tries sources in this order:
1. RunPod Object Storage (S3-compatible)
2. Civitai fallback

Set these **environment variables on the endpoint**:

Required for S3-first mode:
- `S3_BUCKET` (example: `u3zpho58es`)
- `S3_ENDPOINT_URL` (example: `https://s3api-us-ca-2.runpod.io`)
- `AWS_ACCESS_KEY_ID`
- `AWS_SECRET_ACCESS_KEY`
- `AWS_DEFAULT_REGION` (example: `us-ca-2`)

Optional:
- `S3_KEY` (default: `models/checkpoints/CyberRealistic_Pony_-_v16-0.safetensors`)

Fallback source:
- `CIVITAI_API_TOKEN` (or `CIVITAI_API_KEY`)

## Important disk note
The checkpoint is about 6.6 GB. The container still needs enough local disk to store it in
`/comfyui/models/checkpoints/`. If startup logs show `Errno 28 No space left on device`,
increase endpoint disk size in RunPod Serverless settings.
